Continuous delivery는 코드 변경 사항이 자동으로 빌드, 테스트되고 프로덕션 릴리스를 위해 준비되는 소프트웨어 개발 방식입니다.

기업들은 중간에 개입하거나 클러스터에 수동으로 변경 사항을 적용할 필요 없이, continuous delivery 메커니즘을 통해 EKS 클러스터의 애플리케이션에 대한 변경 사항을 자동화할 수 있습니다.

이 섹션에서는 사용자가 소스 코드 관리, 이미지 생성 및 AWS Continuous delivery 서비스(CodePipeline)를 통한 EKS 클러스터로의 배포까지 소프트웨어 개발 수명 주기를 구성하는 방법을 살펴보겠습니다.
