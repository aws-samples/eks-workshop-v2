컴퓨팅 오토스케일링 개념을 확립했으므로, 이제 클러스터에서 리소스를 확장하고 워크로드를 관리하는 다양한 방법을 종합적으로 살펴볼 수 있습니다. 오토스케일링을 위한 세 가지 주요 메커니즘을 다룰 것입니다:

- Horizontal Pod Autoscaler (HPA)
- Cluster Proportional Autoscaler (CPA)
- Kubernetes Event-Driven Autoscaling (KEDA)

이 장에서는 Horizontal Pod Autoscaler (HPA), Cluster Proportional Autoscaler (CPA) 및 Kubernetes Event-Driven Autoscaling (KEDA)를 구성하고 사용하여 클러스터의 워크로드를 관리하는 프로세스를 살펴보겠습니다.
